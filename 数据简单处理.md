### 3）异常值处理
除了重复数据外，还存在非重复情况下的异常数据，例如同一车辆在**极短时间内**（如40秒）载客状态迅速从0->1->0切换，这在实际业务中不合逻辑。

**需求8 & 9：** 使用Pandas的`shift`函数进行前后对比，筛选出短时间内的状态突变的异常数据。
```python
#上下平移一个单位
df['status_up'] = df['status'].shift(1)      # 向下移动1
df['status_down'] = df['status'].shift(-1)   # 向上移动1
df['id_up'] = df['id'].shift(1)
df['id_down'] = df['id'].shift(-1)
df['time_up'] = df['time'].shift(1)
df['time_down'] = df['time'].shift(-1)

# 筛选条件
cond_1 = (df['status'] != df['status_down'])
cond_2 = (df['status'] != df['status_up'])
cond_3 = (df['id'] == df['id_up'])
cond_4 = (df['id'] == df['id_down'])
cond_5 = ((df['time_down']- df['time_up']).dt.seconds < 60)

df_abn = df[cond_1&cond_2&cond_3&cond_4&cond_5].reset_index()
```

> **简单描述**：**图11** 展示了利用`shift`函数创建的辅助列。红框中可以看到，某条记录`status=1`时，其上下相邻记录的`status`均为0，且时间差非常短，这种“0-1-0”模式会被代码逻辑成功标记为异常。
![图11：通过shift检测异常](img_11)

> **简单描述**：**图1** 展示了检测到的异常数据在各车辆`id`上的分布情况。其中，车辆`28159`出现了高达70次的异常，说明该车辆可能存在较为频繁的传感器问题。
![图1：异常数据id分布](img_1)

**需求11：** 通过判断状态差（当前状态-上一状态），捕捉上车（差值为1）和下车（差值为-1）的位置与时间。
```python

df['status_chg'] = df['status'] - df['status_up']
df['id_chg'] = df['id'] - df['id_up']
# 筛选上车和下车的标识1和-1，并为同一车辆
df_temp = df.loc[((df['status_chg']==1) | (df['status_chg']==-1)) & (df['id_chg']==0)]

# 将时间、经纬度向上偏移1个单位，拼接成一行
df_temp['Etime'] = df_temp['time'].shift(-1)
# ...（后续代码用于修改列名并筛选）
```
