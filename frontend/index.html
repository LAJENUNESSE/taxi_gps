深圳出租车GPS数据分析系统

完整流水线：数据清洗 → OD提取 → 数据分析 → 可视化 → 预测 → 百度地图交互 → 路网拥堵 → ETA。本门户整合 4 个交互地图页面与 8 个阶段路线图。

系统调用主线

数据读取与清洗 → OD 提取 → 车辆缓存 / 分钟缓存 / OD 缓存 → 轨迹查询 / 车辆位置查询 → 热力图 / 统计分析 → 路网校正 / HMM 匹配 → 拥堵道路颜色 → ETA 预测 → 系统集成

访问与日志要求

  • 所有交互页面基于百度地图 JavaScript API，受浏览器安全策略限制，不能使用 file:// 协议打开。
  • 在项目根目录启动 python -m http.server 8080，浏览器访问 http://localhost:8080/frontend/index.html。
  • 每日正式提交结果为项目日志，包括完成内容、成员分工、检查点结果、文件位置、问题汇报和次日计划。
  • 代码、数据文件、截图和系统页面不作为每日单独提交材料，但需要能在检查点现场打开、运行或展示。
