运行方式

  • 全量流水线：依次执行 01_数据清洗.py → 02_OD提取.py → 03_数据分析.py → 04_可视化.py → 05_预测.py → 06_百度地图流向.py → 07_缓存构建.py → 08_轨迹查询.py → 09_交互热力图.py → 10_路网分析.py。
  • 演示模式：python -m http.server 8080，浏览器打开 http://localhost:8080/roadmap/demo.html。
  • 或单页直接访问：http://localhost:8080/output/figures/road_congestion.html 等四个页面。

难点

  • 10 个脚本间存在隐式依赖（缓存偏移、路网 GeoJSON、速度统计），任意一步重跑都可能影响下游。
  • 四个百度地图页面均依赖浏览器端 AK 与 HTTP 服务器，验收现场需提前确认网络与端口可用。
  • 全量数据流水线运行时长累计超过 1 小时，验收需提前走通或准备离线产物快照。
  • 文档与代码需双向对齐：脚本路径、输出路径、表格内容任一不一致都会被验收扣分。

注意事项

  • 验收前按 README 运行命令块从头跑一遍，确认所有产物文件均存在且可被对应 HTML 正常加载。
  • 演示门户与阶段入口的所有 href 链接需做相对路径校对，避免本地能开、移交后断链。
  • 百度地图 AK 白名单需包含演示域名；离线演示需提前准备 fallback 或截图。
  • 提交材料应包含：完整代码、数据缓存路径说明、四个交互 HTML、README、roadmap 全套页面。
