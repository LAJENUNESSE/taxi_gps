07 HMM、拥堵道路与 ETA

在路网 snapping 基础上完成路段平均速度统计、拥堵等级配色与百度地图拥堵道路可视化，并衔接 ETA 预测。

返回总入口上一阶段下一阶段

完成内容

  • 执行 src/10_路网分析.py 第 2 阶段：基于已 snapping 的 GPS 点，按 (u, v, key) 累积每条道路边的车速样本。
  • 计算道路边平均速度，按阈值划分拥堵等级：畅通 > 45 km/h（绿 #228B22）、缓行 20–45 km/h（黄 #FFD700）、拥堵 < 20 km/h（红 #DC143C）。
  • 导出 road_speeds.json，记录每条边的样本数、平均速度与拥堵等级，供 HTML 与 ETA 模型复用。
  • 生成 road_congestion.html（百度地图 JS API）：按拥堵等级着色的道路线段图层，叠加导航/比例控件。
  • ETA 衔接：路段平均速度作为 05_预测.py XGBoost ETA 模型的特征来源之一，形成“路网感知—速度统计—ETA”闭环。

输出文件

  • data/road_network/road_speeds.json — 道路边速度与拥堵等级。
  • output/figures/road_congestion.html — 百度地图路网拥堵可视化页面。

运行方式

  • python src/10_路网分析.py --serve --port 8080 — 生成 HTML 并启动 HTTP 服务器。
  • 浏览器打开 http://localhost:8080/road_congestion.html。
  • 需先完成 06 阶段的路网下载与 snapping，否则 road_speeds.json 为空。

难点

  • HMM 地图匹配需考虑发射概率（GPS 点到边的距离）与转移概率（相邻点边的连通性），纯最近邻 snapping 在路口容易出错。
  • 样本数过少的边速度不可信，需设置最小样本阈值，否则单点高速会被误判为畅通。
  • 百度地图 Polyline 数量受性能限制，需对道路边按速度分批渲染或做矢量合并。
  • ETA 模型依赖路段速度，若速度统计口径与预测特征不一致会引入偏差。

注意事项

  • 必须通过 HTTP 服务器访问，file:// 无法加载百度地图 API。
  • 拥堵阈值（45 / 20 km/h）写在脚本常量区，调整后需重跑统计与 HTML 生成。
  • 颜色常量 COLOR_LOW / COLOR_MEDIUM / COLOR_HIGH 需与轨迹回放页速度配色一致，避免视觉语义冲突。
  • 路网可视化点数大，建议开启 enableMassClear 与按缩放聚合，缓解低端设备卡顿。
