04 地图基础与轨迹查询

基于百度地图 JavaScript API 实现交互式轨迹回放查看器，支持速度渐变折线、上下客标记与信息面板。

返回总入口上一阶段下一阶段

完成内容

  • 执行 src/08_轨迹查询.py，从 2.6 GB vehicle_data.json 流式抽取 100 辆代表性车辆轨迹。
  • 分层抽样选取车辆：兼顾轨迹长/短、订单多/少，剔除无订单的车辆（无法展示上下客点）。
  • 单车轨迹点上限 1500，超出则均匀降采样，控制 HTML 体积。
  • 搭配 orders.csv 抽取每辆车对应的上下客事件，作为地图标记。
  • 生成 trajectory_viewer.html（百度地图 JS API v3.0）：速度渐变折线、回放动画、上下客标记、实时信息面板。
  • 生成数据文件 trajectory_sample.json，存放在 output/figures/，供 HTML 异步加载。

输出文件

  • output/figures/trajectory_viewer.html — 交互式轨迹回放页面（百度地图）。
  • output/figures/trajectory_sample.json — 抽样轨迹与上下客事件数据。

运行方式

  • python src/08_轨迹查询.py — 生成数据 + HTML。
  • python src/08_轨迹查询.py --serve — 生成后启动 HTTP 服务器。
  • 浏览器打开 http://localhost:8080/trajectory_viewer.html。
  • 注意：必须通过 HTTP 服务器访问，file:// 会被浏览器安全策略拦截，无法加载百度地图 API。

难点

  • vehicle_data.json 太大不能整体加载，必须按车辆键定位偏移、随机读取对应片段。
  • 百点点轨迹若不做降采样，HTML 会膨胀到数十 MB，浏览器卡顿；需在保真度与体积间平衡。
  • 上下客标记时间与轨迹点时间不一定精确对齐，需用时间最近邻匹配方能落到轨迹上。
  • 速度渐变配色需与 10_路网分析.py 的拥堵色阶一致（绿/黄/红），否则同一系统内语义混乱。

注意事项

  • 百度地图 AK 已在 src/config.py 配置，HTML 内为浏览器端 AK，注意域名白名单。
  • 轨迹回放动画性能依赖点数，建议单次回放 ≤ 1500 点。
  • trajectory_sample.json 是离线数据，重新抽样需重跑脚本，不要手动编辑。
  • 页面内控件应使用百度地图原生 API 而非第三方库，减少依赖加载时间。
