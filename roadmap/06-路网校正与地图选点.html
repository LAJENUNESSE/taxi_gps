06 路网校正与地图选点

完成深圳路网加载与 GPS 轨迹到道路边的几何 snapping，为路网级拥堵可视化与 HMM 地图匹配打基础。

返回总入口上一阶段下一阶段

完成内容

  • 执行 src/10_路网分析.py 第 1 阶段：从 OpenStreetMap 下载深圳 drive 路网，保存为 GeoJSON。
  • 离线 fallback：若 OSM 不可达，生成覆盖深圳边界的网格路网（_create_synthetic_fallback），保证流程不中断。
  • 产出 shenzhen_edges.geojson（道路边）与 shenzhen_nodes.geojson（节点），含 osmid / u / v / key / geometry / highway 字段。
  • 流式读取 2.6 GB vehicle_data.json（不载入全量内存），按 ~3% 采样约 1.1M GPS 点。
  • 用 scipy.spatial.cKDTree（K=5 候选）+ shapely 精确距离选出最近道路边，将 GPS 点 snapping 到路网。
  • 路网选点：地图页面提供深圳中心 (114.06, 22.55) 与默认 zoom 12 作为统一视角。

输出文件

  • data/road_network/shenzhen_edges.geojson — 深圳路网边（LineString 集合）。
  • data/road_network/shenzhen_nodes.geojson — 路网节点（Point 集合）。
  • data/road_network/road_speeds.json — 按 (u,v,key) 累积的车速统计（待下阶段填充拥堵等级）。

运行方式

  • python src/10_路网分析.py — 默认流式读 vehicle_data.json，下载路网并完成 snapping + 速度累积。
  • python src/10_路网分析.py --use-clean-csv — 从 clean.csv 采样（更快，无需先建缓存）。
  • 依赖 osmnx / geopandas / shapely / pyproj / scipy，需提前安装。

难点

  • OSMX 在受限网络环境下下载 OSM 容易超时失败，必须提供 fallback 网格路网。
  • GPS 点 snapping 到道路边是几何近似，遇高架、平行路容易选错边，需结合 heading 或速度方向做二次校验。
  • 2.6 GB 轨迹缓存不能整体加载，需用正则 _VEHICLE_KEY_RE 流式定位车辆键后再读片段。
  • cKDTree 在数万条边上构建较慢，需缓存索引避免每次重跑。

注意事项

  • 路网 GeoJSON 体积较大，统一存放 data/road_network/，不要混入 output/。
  • 采样率 SAMPLE_RATE=0.03 与随机种子 42 固定，便于结果复现。
  • 路网易随 OSM 上游更新而变化，固化一次后不要随意重下，避免速度统计基准漂移。
  • 距离阈值需设置上限，否则离路较远的 GPS 点会被强行吸附到最近边，引入噪声。
