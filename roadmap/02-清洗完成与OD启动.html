02 清洗完成与 OD 启动

完成分块读取和关键清洗，并开始识别上车点、下车点。

返回总入口上一阶段下一阶段

完成内容

  • 使用 pd.read_csv(..., chunksize=...) 完成大文件分块读取。
  • 完成字段统一，建议统一为 id,time,long,lati,status,speed 或 id,time,lng,lat,status,speed，但全项目必须保持一致。
  • 如果原始时间只有时分秒，需要拼接日期，例如 2013-10-22 后再转为 datetime。
  • 完成排序：按 id,time 升序排序并重置索引。
  • 统计 id,time 重复数据，处理重复数量为 2 和 3 的情况。
  • 使用 shift 构造前后状态、前后车辆 ID、前后时间，识别短时间异常状态。
  • 输出清洗后的数据文件，并记录清洗前后数据量。
  • 开始 OD 提取：计算 status_chg，初步筛选 0 -> 1 上车点和 1 -> 0 下车点。

难点

  • 不能简单使用 drop_duplicates 处理所有重复记录；同一车辆同一时间可能有状态相同、状态不同、重复数量为 2 或 3 等情况。
  • 分块读取时，同一辆车的数据可能跨分块，状态判断需要保留边界记录。
  • 异常值既可能来自重复数据，也可能来自非重复状态跳变。
  • OD 启动依赖排序和状态清洗，如果清洗不稳定，后续订单表会产生大量错误订单。

注意事项

  • 清洗规则必须写入项目日志，不能只保留代码。
  • 清洗输出要有固定命名和路径，供 OD 提取直接调用。
  • 检查点需要能展示字段类型、重复数量、异常数量、清洗前后行数，以及上车点/下车点的初步数量。
  • 不要在代码中保留个人绝对路径；应把输入、输出路径集中放在配置区。
  • 如果读取旧代码中的中文列名出现乱码，应尽早改为统一英文列名，避免后续热力图和机器学习阶段反复重命名。
