05 热力图与统计分析

基于百度地图 HeatmapOverlay 实现按小时切换的上客点交互热力图，并完成载客率、里程与订单统计。

返回总入口上一阶段下一阶段

完成内容

  • 执行 src/09_交互热力图.py，读取 orders.csv 上客点（开始经度/开始纬度/开始时间）。
  • 按小时分桶 + ~100 m 网格聚合计数（深圳纬度下 0.001° ≈ 111 m），输出 {hour: [[lng, lat, count], ...]}。
  • 越界过滤（深圳范围）并解析时间提取小时，剔除无效上客点。
  • 生成 interactive_heatmap.html（百度地图 JS API + HeatmapOverlay）：时间滑块 0–23h、热力图层、图例、信息面板、导航/比例控件。
  • 每小时最大点数 20000 上限，避免单小时网格过多撑爆 HTML。
  • 同步复用 03_数据分析.py 的统计结果（载客率、里程、订单时段分布），供静态图与交互热力图互为印证。

输出文件

  • output/figures/interactive_heatmap.html — 上客点交互热力图（按小时切换）。
  • 配套静态图：static_heatmap.png、heatmap_slices.png、dynamic_heatmap.gif（由 04_可视化.py 产出）。

运行方式

  • python src/09_交互热力图.py — 仅生成 HTML。
  • python src/09_交互热力图.py --serve — 生成 + 启动 HTTP 服务器。
  • python src/09_交互热力图.py --serve --port 8000 — 指定端口。
  • 浏览器打开 http://localhost:8080/interactive_heatmap.html。

难点

  • 上客点近 48 万个，直接铺到地图会卡死，必须先做空间聚合再交给 HeatmapOverlay。
  • 网格大小需平衡精度与性能：0.001° (~111 m) 在深圳主城区合理，但郊区点稀疏会显得空旷。
  • 百度地图 HeatmapOverlay 对点数有上限，单小时超过 2 万点需二次抽样或减小最大半径。
  • 时间滑块切换需重新设置热力数据，避免内存泄漏（旧数据未释放）。

注意事项

  • 必须通过 HTTP 服务器访问，file:// 无法加载百度地图 API 与 HeatmapOverlay。
  • 热力图配色阈值需根据最大计数值动态调整，否则低峰小时几乎看不见热区。
  • 上客点过滤范围必须与 SHENZHEN_BOUNDS 一致，否则会出现落在水域或边界外的异常点。
  • 统计口径应与 stats_summary.json 一致，避免交互图与静态图数据打架。
